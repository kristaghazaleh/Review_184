Core Facts

This is the compact core of the page. These facts show up repeatedly and connect directly to the larger reasoning problems.

Sampling / Antialiasing

  • Nyquist: sample above twice the highest frequency to avoid aliasing.
  • Aliasing is frequency folding after undersampling.
  • Antialiasing = low-pass filter before sampling, or supersample then downsample.
  • Supersampling improves edge coverage and fine detail.
  • You do not resolve each triangle independently into the final framebuffer.

Transforms / Pipeline

  • Order matters: matrix multiplication is not commutative.
  • Translation requires homogeneous coordinates.
  • Perspective involves a division.
  • Z-buffer stores closest depth seen so far.
  • Object \(\rightarrow\) world \(\rightarrow\) camera \(\rightarrow\) clip \(\rightarrow\) NDC \(\rightarrow\) screen.

Textures / Mipmaps

  • Bilinear = interpolation within one mip level.
  • Trilinear = bilinear in two mip levels + interpolation across levels.
  • Smaller screen footprint with same UV stretch usually means higher mip level.
  • Total mipmap storage is about \(\frac{4}{3}\) of the base level.
  • Level choice depends on UV footprint, not just triangle size alone.

Curves / Splines

  • Cubic Bézier uses 4 control points.
  • de Casteljau for one cubic evaluation computes 6 intermediate lerps.
  • Bézier curve stays in convex hull of control points.
  • Hermite uses endpoint positions and endpoint tangents.
  • Catmull-Rom interpolates through interior control points.

Meshes / Subdivision

  • After \(k\) Loop subdivisions, triangle count becomes \(N \cdot 4^k\).
  • Half-edge operations are local connectivity rewrites.
  • Area-weighted normals average adjacent face normals.
  • Edge collapse in a triangle manifold interior removes two faces.
  • Subdivision smooths but does not preserve exact shape.

Ray Tracing / MC

  • Ray–sphere intersection leads to a quadratic.
  • BVH nodes store a box containing all geometry below them.
  • Irradiance units: \(W / m^2\).
  • Hemisphere solid angle is \(2\pi\) steradians.
  • Monte Carlo estimators can be unbiased but noisy; importance sampling lowers variance when matched well.

Topic 1: Sampling, Aliasing, and Antialiasing

Core idea

A continuous signal can contain frequencies too high for a given sampling rate to represent faithfully. When that happens, those frequencies masquerade as lower frequencies after sampling. That is aliasing. The Nyquist principle says that if the highest frequency present is \(f_{max}\), then the sampling frequency must exceed \(2f_{max}\) to avoid aliasing.

What to recognize on an exam

  • If the question mentions jaggies, moiré, wagon-wheel effects, or repeated wrong patterns, think aliasing.
  • If it asks how to reduce aliasing, think prefiltering, supersampling, or better reconstruction.
  • If it asks why supersampling helps, think more sample points capture more of the underlying variation before averaging.

Rasterization and supersampling

In basic rasterization, each pixel is tested against geometry using one or more sample points. In supersampling, each pixel contains several subsamples. You rasterize into that higher-resolution sample grid first, then resolve to a final color by averaging down. That is why supersampling is conceptually closer to “sample more densely, then low-pass” than to “blur afterward.”

Useful mental sentence: supersampling does not mean “finish one triangle completely into the final framebuffer, then move on.” All triangles should contribute to the sample buffer before the resolve step.

Inside tests and barycentric thinking

For triangles, point-in-triangle tests often use edge functions or barycentric coordinates. A point is inside only if the barycentric coordinates satisfy the full inside condition, not merely that they sum to 1. Summing to 1 is necessary for affine combination, but not sufficient for inside-ness; you also need each coordinate to be nonnegative when using the standard convention for points inside the triangle.

Common mistakes

  • Confusing aliasing with blur. Blur can reduce aliasing, but blur is not aliasing.
  • Thinking Nyquist uses “twice the number of samples” in some vague sense. It is specifically about frequency content.
  • Thinking “barycentric coordinates sum to 1” alone implies the point is inside a triangle.
  • Resolving each primitive separately instead of accumulating all primitive contributions before the downsample.

Mini checklist for any sampling question

  1. What is being sampled: image, texture, time signal, geometry coverage?
  2. What is the relevant frequency content?
  3. Is the sampling rate high enough?
  4. If not, what filtering or denser sampling can reduce aliasing?

Topic 2: Transforms, Homogeneous Coordinates, and the Graphics Pipeline

Core idea

Graphics transforms let us move between coordinate systems. The most important habits are: keep track of what space you are in, remember that order matters, and know that translation and perspective need homogeneous coordinates.

Standard matrix forms

2D translation: [1 0 tx] [0 1 ty] [0 0 1] 2D rotation: [ cos θ -sin θ 0 ] [ sin θ cos θ 0 ] [ 0 0 1 ] 2D scale: [sx 0 0] [0 sy 0] [0 0 1]

Why homogeneous coordinates matter

Linear \(2 \times 2\) or \(3 \times 3\) matrices alone cannot represent translation of points as a pure matrix multiply in Cartesian coordinates. Homogeneous coordinates add one extra coordinate so translation becomes linear in the higher-dimensional representation. Perspective projection also ends with a division, which is why “perspective requires division” keeps appearing in the old exams.

Order of operations

Matrix products are applied right-to-left to column vectors. So if you write \(M = T R S\), then the point is first scaled, then rotated, then translated. This is where many exam mistakes come from. Rotating then translating is usually not the same as translating then rotating. Scaling then rotating is also not the same as rotating then scaling unless the scaling is uniform and the circumstances make the operations commute.

Fast recognition cues

  • If a picture shows the object staying attached to a particular corner or pivot, the transform probably includes translate to pivot, rotate/scale, translate back.
  • If a matrix has a last column with nonzero entries in homogeneous form, it includes a translation component.
  • If the question says “frame-to-world” versus “world-to-frame,” think inverse transforms and reversed order.

Pipeline sequence

A clean mental model is:

object space \(\rightarrow\) world space \(\rightarrow\) camera / view space \(\rightarrow\) clip space \(\rightarrow\) perspective divide \(\rightarrow\) normalized device coordinates \(\rightarrow\) screen / raster space

Z-buffer essentials

The Z-buffer stores the closest depth encountered so far for each pixel or sample. At the start of a new frame, you initialize it to the farthest possible depth, not zero by default. Then each new fragment updates the color and Z value only if it is closer than the stored value.

Common mistakes

  • Forgetting whether a matrix acts in world-to-camera or camera-to-world direction.
  • Multiplying in the wrong order or reading the order left-to-right when using column vectors.
  • Thinking perspective shrinkage goes like \(1/d^2\) instead of roughly \(1/d\) in image size.
  • Resetting the Z-buffer to zero instead of “infinitely far.”

Topic 3: Texture Mapping, Filtering, and Mipmaps

Core idea

Texture mapping asks how image data in texture space \((u,v)\) should be sampled when projected onto the screen. The central issue is that one screen sample often corresponds to a footprint covering more than one texel, especially for minification. That is why texture mapping is fundamentally another sampling problem.

Nearest, bilinear, trilinear

  • Nearest: choose the closest texel center.
  • Bilinear: interpolate among the four neighboring texels within one mip level.
  • Trilinear: do bilinear in two adjacent mip levels and interpolate between the two results.

Mipmap logic

Mipmaps precompute lower-resolution versions of the same texture so minification can be filtered more appropriately. The total storage cost is a geometric series:

\(1 + \frac14 + \frac1{16} + \cdots = \frac{4}{3}\)

So full mipmaps use about \(\frac43\) the memory of the base texture level.

How level selection works conceptually

The chosen mip level depends on the size of the pixel footprint in texture space. If one screen pixel covers a large region of UV space, then the renderer should use a coarser mip level. In practical implementations, that footprint is estimated using derivatives such as \(du/dx, dv/dx, du/dy, dv/dy\), or equivalently nearby UV samples. Bigger footprint means bigger level.

What to recognize on an exam

  • If the object gets smaller on screen but its texture coordinates span the same range, expect a higher mip level.
  • If the question contrasts bilinear and trilinear, remember that only trilinear blends across levels.
  • If it asks why mipmaps help, answer in the language of antialiasing during minification.

Common mistakes

  • Saying bilinear includes interpolation across mip levels.
  • Thinking mip level depends only on triangle area in screen space; it really depends on the UV footprint relative to the screen footprint.
  • Forgetting that magnification and minification are different regimes.

Topic 4: Curves, Splines, Bézier Surfaces, Half-Edge Meshes, and Subdivision

Bézier curves and de Casteljau

de Casteljau evaluates a Bézier curve by repeated linear interpolation. For a cubic Bézier curve with control points \(P_0, P_1, P_2, P_3\), one evaluation performs three first-level lerps, then two second-level lerps, then one final lerp: six interpolated points total.

\(Q_i = (1-t)P_i + tP_{i+1}\)

Repeating this process until one point remains gives the point on the curve at parameter \(t\). The curve stays within the convex hull of its control points.

Hermite and Catmull-Rom

  • Cubic Hermite: specified by two endpoint positions and two endpoint tangent vectors.
  • Catmull-Rom: interpolates through the interior control points of a sequence without requiring tangent vectors to be given directly.

Bézier surfaces

Extend de Casteljau in one parameter direction to get intermediate points, then evaluate again in the other direction. The important exam habit is to explain the idea clearly rather than drowning in notation.

Vertex normals

Smooth shading at a vertex often uses an area-weighted average of neighboring face normals. Each face normal is typically computed by a cross product of two edge vectors, scaled automatically by triangle area. Sum them, then normalize the result.

Half-edge structure

A half-edge stores local mesh connectivity: next, twin, vertex, edge, and face references. Edge flip, split, and collapse are all local rewiring operations. The safest way to reason through them is to label the neighborhood, write down the before-and-after connectivity, and then check every pointer.

Loop subdivision

Loop subdivision refines a triangle mesh by splitting edges, repositioning old and new vertices according to weighted rules, and producing four child triangles from each old triangle. Therefore, if you begin with \(N\) triangles, after \(k\) levels you have \(N \cdot 4^k\) triangles.

Common mistakes

  • Confusing “control point” with “point on the Bézier curve.”
  • Forgetting Hermite needs tangents.
  • Thinking subdivision preserves the exact original shape.
  • Trying to reason about a half-edge operation without drawing the local neighborhood first.

Topic 5: Ray Tracing and Acceleration Structures

Core idea

Ray tracing follows rays through the scene and asks what they hit. That means the whole topic reduces to: write the ray, write the surface equation, substitute, and solve for valid \(t\)-values.

Ray: r(t) = o + t d Sphere with center c and radius R: ||x - c||^2 = R^2 Substitute x = o + t d => quadratic in t

Ray–sphere intersection

Substitute \(r(t) = o + td\) into the sphere equation \(\|x-c\|^2 = R^2\). Expanding gives a quadratic in \(t\). Solve the quadratic, then keep only valid roots in the permitted interval, usually something like \(t_{min} \le t \le t_{max}\). If there are two roots, the smaller positive valid one is the first hit.

BVH intuition

A Bounding Volume Hierarchy groups primitives recursively inside bounding boxes so whole groups can be rejected quickly. A node’s bounding box contains the union of all geometry in its subtree. It is not “the larger of two child boxes” in some simplistic sense; it must enclose both children fully.

What to recognize on an exam

  • If the algebra expands into a quadratic, it is probably a ray–sphere problem.
  • If the question asks why acceleration helps, answer in the language of culling many primitives with one cheap box test.
  • If the question asks about path depth and termination, think of Russian roulette and unbiasedness.

Common mistakes

  • Keeping negative \(t\)-values that lie behind the ray origin.
  • Forgetting to choose the nearest valid root.
  • Describing BVH boxes as if they store only one child or only the larger child.

Topic 6: Radiometry, BRDFs, and Monte Carlo Integration

Radiometric quantities

  • Radiant flux \(\Phi\): power, measured in watts.
  • Irradiance \(E\): incoming power per unit area, measured in \(W/m^2\).
  • Radiance \(L\): power per unit projected area per unit solid angle. This is the core quantity in rendering equations.
  • Illuminance: photometric analogue of irradiance, measured in lux.
  • Solid angle: measured in steradians; a hemisphere subtends \(2\pi\), not \(4\pi\).

Lambertian / diffuse reflection

A Lambertian BRDF is constant with respect to outgoing direction, which is why matte surfaces look directionally uniform. The familiar shading term \(l \cdot n\) comes from Lambert’s cosine law, not from the BRDF alone.

Monte Carlo estimator structure

\[ \int_a^b f(x)\,dx \approx \frac{1}{N}\sum_{i=1}^N \frac{f(X_i)}{p(X_i)} \]

In the uniform case on \([a,b]\), \(p(x)=1/(b-a)\), so the estimator becomes the average sample value times \((b-a)\). That estimator is unbiased. Unbiased means its expected value equals the true integral. It does not mean each individual estimate is close.

Importance sampling

If you choose a PDF \(p(x)\) that places more samples where the integrand contributes most, then variance usually decreases. The estimator remains unbiased as long as you divide by the same PDF you sampled from and the support is correct wherever the integrand is nonzero.

Russian roulette

Russian roulette randomly terminates low-contribution paths to keep computation finite while preserving unbiasedness by compensating surviving paths appropriately. It is a variance / efficiency tradeoff device, not a biased hack to arbitrarily cut off bounces.

Common mistakes

  • Mixing up irradiance and radiance.
  • Saying hemisphere solid angle is \(4\pi\).
  • Confusing unbiasedness with low variance.
  • Forgetting the division by PDF in the Monte Carlo estimator.

Recurring Traps and How to Beat Them

Trap: “I kind of know this topic”

That feeling often hides a direction mistake, a units mistake, or a condition mistake. Fix it by practicing the exact sentence you would write on the exam.

Trap: transform order

Always identify the pivot or reference frame. Ask: what must happen first to make the later operation happen around the right point?

Trap: texture filtering vocabulary

Memorize the pair: bilinear = within one level, trilinear = across two levels.

Trap: barycentric coordinates

Sum to one is not enough. For inside tests, you also need the sign / nonnegativity condition under the usual setup.

Trap: unbiased versus good

An unbiased estimator can still look terrible for one sample count because variance can be large. Keep expected value and variance mentally separate.

Trap: memorizing facts without recognition

Every flashcard should answer one of these: what is this, when do I use it, what is the standard test, what is the common confusion?

Trap: half-edge operations

Never do them purely verbally. Draw the local patch, label everything, and only then reason through the pointer updates.

Trap: Z-buffer initialization

The initial depth should represent “nothing close has been seen yet,” so initialize to the far end, not zero by habit.

Checks and Answers

These prompts are followed immediately by the answer so the page can also serve as a direct reference.

1. What does Nyquist actually say, and what specific failure happens below that rate?

Answer. If the highest frequency present is \(f_{max}\), then the sampling rate must exceed \(2f_{max}\) to avoid aliasing. Below that rate, high frequencies fold into incorrect lower frequencies, so the sampled signal represents the wrong pattern.

2. Why does prefiltering reduce aliasing?

Answer. Prefiltering removes or weakens frequency content that the sampling grid cannot represent. After the high frequencies are attenuated, the remaining signal is less likely to fold into false low-frequency patterns.

3. In supersampled rasterization, why is it wrong to resolve each triangle separately into the final framebuffer?

Answer. Because supersampling works by accumulating coverage and color in a higher-resolution sample buffer first, then averaging once at the end. Resolving each triangle separately throws away the interactions among overlapping primitives and breaks the intended filtering step.

4. What conditions on barycentric coordinates indicate a point is inside a triangle?

Answer. The coordinates must sum to 1 and, under the usual inside test convention, each coordinate must be nonnegative. Sum-to-one alone only says the point is an affine combination; it does not guarantee the point lies inside the triangle.

5. Why can translation not be represented by an ordinary \(2 \times 2\) matrix in 2D Cartesian coordinates?

Answer. A \(2 \times 2\) linear map always sends the origin to the origin, but a translation moves every point by a fixed offset and therefore moves the origin too. Homogeneous coordinates add one extra coordinate so translation becomes a matrix multiply in the larger space.

6. Why does perspective projection require division?

Answer. After projecting into clip coordinates, the coordinates must be divided by the homogeneous \(w\) component to produce normalized device coordinates. That division is what creates the depth-dependent shrinking seen in perspective.

7. How do you read the order of a matrix product acting on a column vector?

Answer. Read it right to left. If \(M = TRS\) and points are column vectors, then the point is scaled first, then rotated, then translated.

8. What is the safest way to infer a transform sequence from a before/after diagram?

Answer. Identify the pivot or reference frame first, then ask what operations must happen around that pivot. When an object rotates or scales about a point other than the origin, the standard pattern is translate to the pivot, apply the rotation or scale, then translate back.

9. What does the Z-buffer store, and what should it be initialized to at a new frame?

Answer. It stores the closest depth seen so far for each pixel or sample. At the start of a frame it should be initialized to the farthest possible depth, so that the first valid fragment at each location passes the depth test.

10. What is the difference between nearest, bilinear, and trilinear filtering?

Answer. Nearest chooses one texel. Bilinear interpolates among the four nearest texels within a single mip level. Trilinear performs bilinear filtering in two adjacent mip levels and then interpolates between those two results.

11. Why do mipmaps help with minification?

Answer. When one screen sample covers many texels, sampling only the base texture causes aliasing. Mipmaps provide prefiltered lower-resolution versions of the texture, so the renderer can sample a representation whose scale better matches the screen-space footprint.

12. Why is total mipmap memory about \(\frac43\) of the base texture?

Answer. Each level has about one quarter as many texels as the previous one, so the total storage is the geometric series \(1 + \frac14 + \frac1{16} + \cdots = \frac43\) times the base level.

13. If a textured object gets smaller on screen, why does the mip level usually go up?

Answer. Because each screen pixel then covers a larger footprint in texture space. A larger UV footprint means the renderer should sample a coarser mip level to match that footprint and reduce aliasing.

14. How many intermediate lerps are computed when evaluating a cubic Bézier point by de Casteljau?

Answer. Six. A cubic Bézier evaluation uses three first-level lerps, two second-level lerps, and one final lerp.

15. Why does a Bézier curve stay inside the convex hull of its control points?

Answer. de Casteljau uses repeated linear interpolation, and every linear interpolation between two points stays on the line segment joining them. Repeating that process keeps every intermediate point, and therefore the final point, inside the convex hull.

16. What four pieces of data define a cubic Hermite segment?

Answer. The two endpoint positions and the two endpoint tangent vectors.

17. What does Catmull-Rom interpolate, and what does it spare you from specifying manually?

Answer. It interpolates through the interior control points of the sequence. It spares you from manually specifying endpoint tangents, because those are derived from neighboring control points.

18. How do you conceptually evaluate a Bézier surface?

Answer. Apply de Casteljau in one parameter direction to produce intermediate points or curves, then evaluate again in the other parameter direction. It is the same repeated interpolation idea used for curves, just applied twice.

19. Why does area-weighting make sense for smooth vertex normals?

Answer. Larger adjacent faces should influence the vertex normal more strongly than tiny faces. Using area-weighted face normals gives a smoother and more geometrically representative average before normalizing.

20. What references does a half-edge usually store?

Answer. A reference to its next half-edge, its twin half-edge, its incident vertex, its parent edge, and its incident face.

21. What is the best procedure for reasoning about edge flip or split on paper?

Answer. Draw the local neighborhood, label every half-edge, vertex, edge, and face, then write the before-and-after connectivity explicitly. After that, check each stored reference one by one.

22. After \(k\) Loop subdivision levels, how many triangles do you have starting from \(N\)?

Answer. You have \(N \cdot 4^k\) triangles, because each old triangle is replaced by four child triangles at every level.

23. Why does subdivision not exactly preserve the original shape?

Answer. Subdivision changes connectivity and also repositions vertices using weighted averaging rules. Those smoothing rules move the mesh toward a smoother limit surface rather than keeping every original feature exactly fixed.

24. What is the first algebraic move in any ray–sphere intersection problem?

Answer. Write the ray as \(r(t)=o+td\), substitute it into the sphere equation \(\|x-c\|^2 = R^2\), and expand. That turns the intersection test into a quadratic equation in \(t\).

25. Once you solve the quadratic for a ray–sphere intersection, how do you choose the physically relevant root?

Answer. Keep only roots in the allowed \(t\)-interval and choose the smallest valid positive root, because that is the first surface hit along the ray.

26. What does a BVH node’s bounding box have to contain?

Answer. It must contain all geometry in that node’s entire subtree, equivalently the union of the geometry contained by both children.

27. Why does a BVH speed rendering up?

Answer. A single cheap ray–box test can reject many primitives at once. When a ray misses a bounding box, the renderer skips every primitive inside that subtree.

28. What are the units of irradiance?

Answer. Watts per square meter, written \(W/m^2\).

29. What is the solid angle of a hemisphere?

Answer. \(2\pi\) steradians.

30. What does the \(l \cdot n\) term represent physically?

Answer. It is the cosine factor from Lambert’s cosine law. It measures how obliquely the incoming light strikes the surface, so it accounts for the projected area seen by the light direction.

31. What does it mean for a Monte Carlo estimator to be unbiased?

Answer. Its expected value equals the true quantity being estimated. Over many repeated trials, the average estimate converges to the correct value.

32. Why can an unbiased estimator still produce a noisy image?

Answer. Because unbiasedness says nothing about variance. An estimator can be correct in expectation while individual finite-sample estimates fluctuate a lot.

33. Why does importance sampling reduce variance when chosen well?

Answer. Because it draws samples more often from regions that contribute more to the integral. That spends computation where the integrand matters most and usually reduces wasted samples.

34. Why do you divide by the PDF in Monte Carlo integration?

Answer. Because the samples were drawn according to that PDF. Dividing by the sampling probability corrects for the nonuniform sampling so the estimator targets the original integral rather than the sampling distribution itself.

35. What does Russian roulette preserve, and what does it trade off?

Answer. It preserves unbiasedness when implemented with the proper survival-probability compensation. It trades reduced average computation for increased variance.

36. Give one sentence distinguishing radiance from irradiance.

Answer. Irradiance is incoming power per unit area, while radiance is power per unit projected area per unit solid angle and is the directional quantity that appears in the rendering equation.
